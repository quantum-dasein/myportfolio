## Architecture Roadmap

1. Launch the current canonical English site and submit `/sitemap.xml` to Google Search Console and Bing Webmaster Tools.
2. Build real static `/de/` and `/en/` routes. Each language must render translated HTML on the server and use reciprocal `hreflang`; the current client-side language switch alone is not enough for German discovery.
3. Add focused, substantial service pages instead of keyword-stuffed doorway pages:
   - `/de/digital-marketing-wien`
   - `/de/technical-seo-wien`
   - `/de/creative-development-wien`
   - `/en/digital-marketing-vienna`
4. Publish one evidence-led article per month. Reuse project research, technical decisions and measured results rather than generic advice.
5. Keep Bridge Consult and FIDIC case metrics updated with date ranges, methodology and links to live work.

## Measurement

- Verify the domain in Google Search Console and Bing Webmaster Tools.
- Track impressions, clicks, indexed pages, branded searches and queries by country.
- Use GA4 only as behavioral analytics; Search Console is the source for organic search visibility.
- Review Core Web Vitals after deployment, especially WebGL pages on mobile.
- Review index coverage and structured-data reports after every major release.

## Launch Checklist

- Production domain resolves consistently to HTTPS and one hostname.
- Sitemap and robots return HTTP 200.
- No staging or preview domains are indexable.
- Every public page has one title, one description, one canonical and one visible H1.
- Social previews use a dedicated 1200 x 630 image.
- Video posters and MP4 files return HTTP 200 and support byte-range requests.
- Contact details are crawlable text and all internal links work without JavaScript.
- German routes are added before targeting German-language keyword clusters at scale.
